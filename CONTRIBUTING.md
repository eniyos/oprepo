# Contributing to OpRepo

Thanks for your interest in contributing!

## How to Contribute

1. **Find an issue** — Check the Issues tab for `good first issue` or `help wanted` labels.
2. **Comment** — Express interest and ask clarifying questions.
3. **Fork & Branch** — Create a feature branch off `main`.
4. **Code** — Follow existing patterns and style.
5. **Test** — Ensure tests pass (`pnpm test`).
6. **PR** — Open a pull request with a clear description.

## Development Setup

See [README.md](./README.md) for the full setup guide.

## Code Style

- TypeScript — strict mode, NestJS conventions for the API
- React — Server Components by default, `'use client'` where needed
- Python — PEP 8, 100 char line limit
- Formatting — Prettier (run `pnpm format`)

## Commit Messages

[Conventional Commits](https://www.conventionalcommits.org/) encouraged:
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `refactor:` code change without feature/fix
- `chore:` tooling, CI, dependencies

## Code of Conduct

Be respectful, inclusive, and constructive. Harassment of any kind will not be tolerated.
